# Установка и подготовка GitHub

## Требуемое ПО

### Для конечных пользователей

- Docker Desktop на Windows
- Docker Engine + Docker Compose на Linux

### Для контрибьюторов

- Python 3.10+
- Node.js 18+
- Docker

## Стандартная процедура запуска

1. Клонируйте репозиторий
2. Создайте `backend/.env` на основе `.env.example`
3. Выполните `docker compose up --build`
4. Откройте `http://127.0.0.1:3000`

## Запуск без вспомогательных скриптов

### Linux

```bash
cp .env.example backend/.env
docker compose up --build
```

### Windows

```bat
copy .env.example backend\.env
docker compose up --build
```

## Настройка LLM-рекомендаций

По умолчанию проект настроен на использование OpenAI ChatGPT-совместимого API.

Ключ доступа нужно поместить в `backend/.env`:

```env
OPENAI_API_KEY=sk-your-key
```

Endpoint и модель задаются в `backend/config/recommendations.json`.

### OpenAI по умолчанию

Оставьте стандартный `api_base_url` и при необходимости замените только модель:

```json
{
  "api_base_url": "https://api.openai.com/v1/chat/completions",
  "model": "gpt-5-nano"
}
```

### Локальная модель через OpenAI-compatible endpoint

Если вы используете локальный сервер, например LM Studio, vLLM, LocalAI или другой совместимый слой, укажите локальный endpoint и имя модели:

```json
{
  "api_base_url": "http://127.0.0.1:1234/v1/chat/completions",
  "model": "local-model-name"
}
```

Если локальный сервер не требует ключа, можно оставить в `backend/.env` любое непустое значение `OPENAI_API_KEY`, если ваш сервер игнорирует заголовок авторизации.

### Другие поставщики моделей

Если другой провайдер предоставляет OpenAI-compatible Chat Completions API, задайте его endpoint и идентификатор модели:

```json
{
  "api_base_url": "https://provider.example.com/v1/chat/completions",
  "model": "provider-model-name"
}
```

Ключ этого провайдера также помещается в `backend/.env` как `OPENAI_API_KEY`.
