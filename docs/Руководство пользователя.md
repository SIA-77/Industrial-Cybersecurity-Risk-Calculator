# Как использовать

## Перед началом

Подготовьте следующее:

- заполненный технический опросник в формате CSV
- опциональный организационный опросник в формате CSV
- список уровней защиты с базовыми значениями PFD
- предположения по нарушителю
- оценочные значения single loss expectancy для каждого события

Используйте шаблоны опросников, поставляемые в:

- `frontend/public/questionnaires/`

## Рабочий процесс

### 1. Запуск приложения

Выполните:

```bash
docker compose up --build
```

Откройте `http://127.0.0.1:3000`.

## 2. Ввод данных

Во вкладке `Input data`:

- скачайте шаблоны опросников (если вы ещё их не заполнили)
- загрузите заполненные CSV-опросники
- выберите тип нарушителя и уровень его возможностей
- настройте интеграцию SIS
- задайте уровни защиты и их значения `pfd`
- отметьте, какие уровни подвержены кибератаке
- введите максимально допустимый ущерб и значения SLE (single loss expectancy) для каждого события

## 3. Проверка диаграммы LOPA

Используйте вкладку `LOPA Diagram`, чтобы убедиться, что последовательность барьеров соответствует предполагаемому сценарию.

Проверьте:

- порядок уровней
- наименования барьеров
- какие уровни являются киберподверженными
- корректно ли отражён финальный путь нарушения

## 4. Запуск оценки риска

Откройте вкладку `Risk assessment`, чтобы вычислить:

- оценку зрелости
- деградированные значения PFD для уровней
- вероятности событий по годам
- ожидаемые потери по событиям

Это основной количественный результат приложения.

## 5. Генерация рекомендаций

Если в `backend/.env` задан `OPENAI_API_KEY`, можно использовать вкладку `Recommendations`.

Бэкенд:

- сформирует структурированный payload на основе оценки
- загрузит требования к отчёту из конфигурации бэкенда
- вызовет настроенную endpoint модели
- вернёт сгенерированные рекомендации

По умолчанию язык отчёта — английский. Для другого языка перед генерацией отчёта укажите дополнительные комментарии для AI.

По умолчанию используется OpenAI ChatGPT-совместимый endpoint из `backend/config/recommendations.json`. При необходимости можно переключиться на другую модель OpenAI, локальный OpenAI-compatible endpoint или другого провайдера с совместимым API, изменив:

- `api_base_url`
- `model`
- `proxy` при необходимости

## 6. Экспорт отчёта

Используйте элемент управления экспортом отчёта для создания:

- PDF
- DOCX

Экспортируемый отчёт включает сводки по опросникам, LOPA-визуализации, таблицы рисков и выводы.

## Требования к файлам

### Опросники

- формат: `.csv`
- максимум заполненных строк: `300`
- ожидаемые столбцы:

`Group`, `Group Name`, `Question ID`, `Question`, `Score`, `Max Score`, `Scoring Guidance`, `Evidence`

### Ввод уровней

Каждый уровень должен содержать:

- `name`
- `pfd` в диапазоне от `0` до `1`

Опционально:

- `cyber`

## Конфигурация

### `.env`

Создайте `backend/.env`:

```env
OPENAI_API_KEY=sk-your-key
```

Примеры конфигурации `backend/config/recommendations.json`:

OpenAI по умолчанию:

```json
{
  "api_base_url": "https://api.openai.com/v1/chat/completions",
  "model": "gpt-5-nano"
}
```

Локальный OpenAI-compatible endpoint:

```json
{
  "api_base_url": "http://127.0.0.1:1234/v1/chat/completions",
  "model": "local-model-name"
}
```

Другой провайдер с совместимым API:

```json
{
  "api_base_url": "https://provider.example.com/v1/chat/completions",
  "model": "provider-model-name"
}
```

## Значения по умолчанию для локальной рабочей станции

Репозиторий защищён для локального использования:

- порты frontend и backend привязаны к `127.0.0.1`
- CORS бэкенда по умолчанию ограничен localhost-источниками
- большие загрузки опросников отклоняются при превышении 300 строк

## Устранение неполадок

### Endpoint рекомендаций недоступен

Проверьте, что:

- файл `backend/.env` существует
- `OPENAI_API_KEY` задан
- настроенный endpoint модели доступен

### В PDF-экспорте есть проблемы со шрифтами

Контейнер бэкенда устанавливает `DejaVu Sans`, а код отчётности регистрирует Unicode-совместимые шрифты. Если вы изменили зависимости, пересоберите образ бэкенда:

```bash
docker compose up --build
```

### Опросник отклоняется

Проверьте:

- расширение CSV
- количество строк менее 300
- наличие обязательных столбцов
- числовые значения в `Score` и `Max Score`
