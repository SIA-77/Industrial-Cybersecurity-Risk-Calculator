# Обзор

## Назначение

ICS Cybersecurity Risk Calculator — это инструмент количественной оценки киберрисков для промышленных систем управления. Его цель — ответить на практический эксплуатационный вопрос:

Как кибератаки изменяют реальный риск промышленных инцидентов, если учитывать уровни защиты и потенциальный бизнес-ущерб?

В отличие от подходов, основанных только на зрелости или только на чек-листах, этот проект преобразует уровень кибербезопасности в измеримые изменения вероятности сценариев и ожидаемых потерь.

## Основная идея

Система объединяет четыре входных блока:

1. Уровни защиты и их базовую вероятность отказа по требованию (PFD)
2. Результаты технического опросника
3. Результаты организационного опросника
4. Профиль нарушителя и предположения по потерям

На основе этих данных бэкенд оценивает, как киберподверженные уровни деградируют под атакой, и пересчитывает итоговые вероятности событий и потери.

## Что отличает этот проект

- Он количественный, а не только качественный
- Он связывает состояние кибербезопасности с промышленной безопасностью и бизнес-ущербом
- Он различает киберподверженные и кибернезависимые барьеры
- Он поддерживает корректировку сценариев с учётом профиля нарушителя
- Он формирует экспортируемые отчёты для инженерных и управленческих процессов

## Предполагаемые пользователи

- команды ICS/OT cybersecurity
- инженеры по промышленной безопасности и надёжности
- владельцы активов и промышленные операторы
- риск-аналитики, подготавливающие сводки для руководства и совета директоров

## Основные возможности в этом репозитории

- разбор опросников из CSV
- количественный расчёт риска с оценкой потерь по событиям
- визуализация уровней в стиле LOPA
- генерация payload для рекомендаций с помощью LLM
- экспорт отчётов в PDF и DOCX

## Позиционирование продукта

Эта документация адаптирована из публичного описания продукта на `icscyberrisk.com` и приведена в соответствие с реализацией в данном репозитории. Open-source версия сохраняет ту же основную концепцию:

- киберриск оценивается в контексте промышленной безопасности
- уровни защиты рассматриваются как измеримые барьеры
- возможности нарушителя влияют на деградацию
- выходные данные предназначены для поддержки реальных решений, а не только для получения меток зрелости
